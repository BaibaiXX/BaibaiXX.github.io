实验题目：利用主流云平台搭建个人博客或网站
实验学时：2 实验日期：2024年3月22日
实验目的：熟悉使用主流云平台并搭建个人博客或者网站。
具体包括：
参考方案：基于主流云平台，设计、实现个人博客或者网站的搭建，撰写实验报告（附带网站链接并可以访问），并在网站上呈现此次实验报告。

硬件环境： 
联网的计算机一台

软件环境：
Windows or Linux

实验步骤与内容：
（1）下载node和npm
（2）安装hexo
打开终端，在终端运行命令：npm install hexo-cli -g #完成hexo的安装
（3）右键打开git bash，在git bash运行命令
hexo init blog
cd blog #进入blog文件
hexo s #生成静态文件
（4）创建github库。建立一个新的库，库名是：username.github.io。我的就是BaibaiXX.github.io。
（5） 创建完成后点击setting里的pages,查看个人网址是否发布。发现没有什么东西，所以先上传了一个html文件，在页面中显示一些内容。然而还是没有，点击github Pages那个蓝色字体，进入链接，在里面递减确认用自己创建的库。发现有自己的网站了。
（6）创建密钥并添加到github账号。由于之前创建过了，所以这一步我跳过。
（7）部署网页。在blog这个文件夹里面，右键打开终端，运行npm install hexo-deployer-git –save，安装发布需要的插件。
（8）在blog文件内先打开_config.yml文件，最下面的deploy部分做如下修改。
deploy:
type: git
repository: ssh://git@github.com:BaibaiXX/BaibaiXX.github.io.git
brandh: main
messge:
这个brandh也可以在github的setting中的Pages进行设置，但是我的已经有main了，所以不需要修改，也可以通过仓库上显示的brandh知道应该写什么。
（9）blog文件夹内打开git bash运行命令，就可以上传到github中了。
hexo s
hexo clean
hexo g
hexo d
（10）打开网站就可以看到内容了，我的网站就是 https://baibaixx.github.io/

结论分析与体会：
（1）首先知道了如何使用hexo创建个人博客。在下载node和npm时，实际上之前我已经下载过了，并且通过node -v和npm -v也显示了对应的版本，但是在今天使用之后，发现node -v还可以显示，但是npm找不到了，所以重新卸载，安装别的版本。然后配置环境就可以了。

（2）虽然搭建好了个人博客，但是在网站上呈现实验报告。我是通过写博客进行上传我的实验报告。如何写博客，通过以下命令进行：
// 命令如下,就可以生成一篇文章
hexo new “标题名称”
hexo g 生成
hexo s 推送
但是在hexo new “标题名称”之后要打开md文件，这个是在source文件夹下的_posts文件夹中，打开md文件进行编辑再执行之后的两个命令。不过要在github中看见的话，需要再执行以下命令：
hexo clean // 用来清除缓存文件,和以生成的静态文件.
hexo g // 生成博客的指令
hexo d // d 表示 depoly[配置,部署的意思]
然后就可以看到了。
